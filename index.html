HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The h1, h2, etc elements represent the level of heading a given text block represents.
  • p elements represent a paragrapgh block of text.
  • ul, ol and li elements represent unordered lists, ordered lists and list items.
  • img elements contain information about images dispalyed on the webpage. This elements contains various attributes; src is the most important. This defines the location of the image.
  • br element creates a line of empty space, or a line break between two blocks of content.
  • a element (which stands for anchor) creates links to the same webpage or others. href attribute points to the URL for the link

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • The most common way to style a webpage is by using an external CSS style sheet. This is preferred in most cases because it allows developers to keep all their CSS rules in a separate file, which makes design changes easier.
  • When using an external style sheet, we link the file using a link element placed in the HTML's head. Add code after the final meta element but before the title element.
  • Declarations contain two important components: the CSS property we want to apply and the value of the property
  • Class attributes as a selector allows us to share a CSS rule to any element we choose. For Ex: using the class selector .card to syle each topic in the Prework Study Guide.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: commit all files in the working branch
  • git commit -m "message*": adds commit message. Short and descriptive of changes made.
  • git pull origin main: pull base(main) branch to our feature branch.
  • git push origin (insert branch name): pushes branch to the main branch.
  • git clone: clone or copy a repo to a new directory or location.
  • git remote -v: verifies connection from local to cloud repo.
  • ls -a: Lists current directory contents

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Declare a variable using var, then followed by a name for it. For example var topics = "HTML, CSS, Git, JavaScript"; array of names or variables listed in quotes are called a string.
  • Conditional statements and loops control the order in wich code is executed.The computer will check if these conditions are truthy. If so, it'll proceed with the action defined in the curly bracket. If not, then the condition will be ignored.
  • If more than one condition using else if will allow multiple conditions till a condtion is truthy and run the code associated.
  • Array is a single variable that is used to hold a group of data. Arrays are typically used to hold data that is related in some way.
  • Index is a unique number assigned to data in an array. In JavaScript and many other programming languages, arrays are zero-indexed and sequential. This means that the first item in any array has an index of 0, not 1
  • Function is a simple set of instructions tat tells the computer how to perform a certain task. Similar to if statements and loops, code that is written in the curly brackets {} will be executed.
  • IMPORTANT A function's name is followed by parentheses ().
